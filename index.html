Thinking in React 2

Never touch the DOM again!
React abstracts the real DOM away from you by utilizing a virtual DOM.
Use React's declarative programming model to tell it what you want to render.
jQuery 

$myBtn = $('button');
$myBtn.addClass('button');
if (toggled) {
  $myBtn.addClass('toggled');
}
$myBtn.appendTo($('body'));

$myBtn.toggleClass('toggled', toggled);
React 

var MyButton = React.createClass({
  render: function() {
    var classes = 'button';
    if (this.state.toggled) classes += ' toggled';

    return <button className={classes}></button>;
  }
});
React + classNames 

var MyButton = React.createClass({
  render: function() {
    var classes = classNames('button', {
      toggled: this.state.toggled
    });

    return <button className={classes}></button>;
  }
});

Thinking in React 3

Invert your data flow
Centralize data ownership and management into as few places as possible.
Pass data into lower-level stateless components via properties.

Thinking in React 4

Embrace composition
Easier to identify components for your application at a mockup.
Pass data into lower-level stateless components via properties.
Smaller, reusable components are easier to test, and easier to reason about.

Thinking in React 5

Beware mutable data
React knows when data has changed.
React relegates data changes to a calls to setState.
Manually mutating data can actually cause some operations to be lost.
